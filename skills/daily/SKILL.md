# /daily — написать daily-лог

Помоги написать daily-лог за сегодня для репозитория <your-qa-repo>.

## Правила стиля

- Максимально кратко и конкретно, от первого лица
- Claude — инструмент, не автор. Не приписывай ему работу
- Не упоминать QA-репозиторий и всё с ним связанное
- Не упоминать TMS и инструменты тестировщика, если они не всплывали в разговоре сегодня
- Не брать пункты из qa playbook, если они не обсуждались сегодня
- В плане на завтра — sub-notes под каждым пунктом, если есть уточнения

## Формат задач в «Дела»

Задачи выносятся в отдельные блоки внутри секции «Дела», каждая на новой строке с живой ссылкой:

```
Завела задачи:
- [Название задачи](https://app.<task-tracker>.com/1/1208919739404549/project/1210659552423846/task/{gid})
- ...

Работала с задачами:
- [Название задачи](https://app.<task-tracker>.com/1/1208919739404549/project/1210659552423846/task/{gid}) — одна фраза что делала
- ...
```

- **Завела задачи** — задачи, созданные тобой сегодня (из <task-tracker>-запроса с `created_by_any`)
- **Работала с задачами** — чужие задачи, с которыми взаимодействовала: оставляла комментарий, расследовала, обновляла описание. Определяется из episodic memory и MemPalace — <task-tracker> API не умеет фильтровать по "комментировал"
- Блок «Работала с задачами» добавляй только если такие задачи были; если нет — пропускай
- Подзадачи в список не включай — только родительские задачи

## Структура недельного файла

Один файл на неделю. Имя: `week-YYYY-MM-DD_MM-DD.md` (понедельник\_пятница текущей недели).

```
# Неделя YYYY-MM-DD / MM-DD

## Понедельник, MM-DD

### Дела

- ...

### Вопросы

- ...

### План

- [ ] ...
  → ...

---

## Вторник, MM-DD

### Дела
...
```

Дни недели по-русски: Понедельник, Вторник, Среда, Четверг, Пятница, Суббота, Воскресенье.

## Процесс

1. Вычисли даты текущей недели (macOS `date`):
   ```bash
   DOW=$(date +%u)  # 1=Пн, 7=Вс
   MONDAY=$(date -v-$((DOW-1))d +%Y-%m-%d)
   FRIDAY=$(date -v+$((5-DOW))d +%Y-%m-%d)
   FRIDAY_SHORT=$(date -v+$((5-DOW))d +%m-%d)
   TODAY_SHORT=$(date +%m-%d)
   WEEKFILE="daily/week-${MONDAY}_${FRIDAY_SHORT}.md"
   ```

2. Выдели, что реально было сделано сегодня.

   **Источники информации (в таком порядке, все обязательны):**
   - Текущий разговор — он уже в контексте, читать его не надо
   - Другие сессии сегодня — ищи через `mcp__episodic-memory__search` с запросом по дате и теме (например `"2026-04-28 <product>"`). НЕ используй `mcp__episodic-memory__read` — он читает файлы целиком и падает с ошибкой "too large"
   - MemPalace diary — `mempalace_diary_read last_n=3`, чтобы не пропустить записи из параллельных сессий
   - <task-tracker> — задачи, созданные сегодня в проекте <product>:
     ```
     mcp__asana__asana_search_tasks(
       workspace="1208919739404549",
       projects_any="1210659552423846",
       created_by_any="1214134527687814",
       created_at_after="<TODAY>T00:00:00Z",
       created_at_before="<TODAY>T23:59:59Z",
       opt_fields="name,created_at"
     )
     ```
     Из результата берём только имена задач. Если задача уже упомянута из других источников — не дублируй.

   **Фильтр для «Дела» — только результаты:**
   - ✅ Что создала: задачи в <task-tracker>, документы, тест-кейсы
   - ✅ Что расследовала / закрыла: баги, инциденты, задачи
   - ✅ Что улучшила / поправила в продакте или процессе
   - ❌ Промежуточные шаги: поиск документов, форматирование файлов, отладка инструментов
   - ❌ Действия Claude: нашёл, прочитал, сформатировал, обновил скилл
   - ❌ Мелочи обслуживания: форматирование <wiki>-страниц, правка заголовков
   - ❌ Внутренняя QA-кухня: обновление скиллов, настройка MCP/хуков, правки схем в <wiki>/MemPalace, чистка окружения — даже если это заняло весь день

   Если что-то кажется результатом, но по сути это шаг — не включай.

3. Составь черновик «План» на следующий день.

   Репо уже клонировано в `/tmp/<your-qa-repo>` (или будет на шаге 4). Прочитай `testing-plan.md`:
   ```bash
   cat /tmp/<your-qa-repo>/testing-plan.md
   ```
   На основе плана предложи 2–4 пункта, которые актуальны на завтра — с учётом того, что уже сделано сегодня и что ещё в очереди. Пункты — конкретные задачи, не категории. Показывай черновик плана пользователю и жди подтверждения или правок перед записью в файл.

4. Если что-то непонятно или сомневаешься — **спроси прежде чем писать**:
   - Стоит ли упоминать конкретное действие?
   - Как точнее сформулировать?
   - Есть ли вопросы или план, о которых ты не знаешь?

5. Клонируй репо (если нет) и запиши/обнови секцию сегодняшнего дня:
   ```bash
   cd /tmp && git clone https://github.com/anastasiiaanfimova/<your-qa-repo>.git 2>/dev/null || true
   cd /tmp/<your-qa-repo> && git pull --quiet
   ```
   - Если файл `$WEEKFILE` не существует — создай с заголовком `# Неделя ${MONDAY} / ${FRIDAY_SHORT}`
   - Если секция сегодняшнего дня уже есть — обнови её
   - Если секции нет — добавь в конец файла, предварив `---`

6. Запушь в GitHub:
   ```bash
   cd /tmp/<your-qa-repo>
   git add $WEEKFILE
   git commit -m "daily: $(date +%Y-%m-%d)"
   git push
   ```

Сегодняшняя дата: используй `currentDate` из контекста или спроси.
